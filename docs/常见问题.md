# 常见问题

## 安装相关

### Q: 如何检查Python版本？

```bash
python --version
# 或
python3 --version
```

需要Python 3.12或更高版本。

### Q: pip安装依赖很慢怎么办？

使用国内镜像源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q: 虚拟环境激活失败？

**Linux/Mac:**
```bash
source venv/bin/activate
```

**Windows:**
```bash
venv\Scripts\activate
```

## 运行相关

### Q: 端口8000被占用怎么办？

在`.env`文件中修改端口：

```
APP_PORT=8080
```

或者找到占用端口的进程并关闭：

```bash
# Linux/Mac
lsof -i :8000
kill -9 <PID>

# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

### Q: 应用启动后无法访问？

1. 检查防火墙设置
2. 确认应用已成功启动
3. 尝试使用 `127.0.0.1:8000` 而不是 `localhost:8000`

### Q: 训练时提示内存不足？

减少训练数据量或关闭其他占用内存的程序。

## 功能相关

### Q: 是否必须配置MongoDB？

不是必须的。如果不配置MongoDB，系统会自动使用本地CSV文件。

### Q: 训练需要多长时间？

使用默认数据约5-10分钟，具体取决于电脑性能。

### Q: 如何使用自己的数据训练？

1. 准备CSV格式的数据文件
2. 确保包含所需的30个特征
3. 在训练页面上传文件
4. 验证数据后开始训练

### Q: 预测结果不准确怎么办？

1. 使用更多高质量的训练数据
2. 检查输入特征是否正确
3. 重新训练模型

### Q: 如何查看训练日志？

训练日志保存在 `logs/` 目录下，文件名格式为 `networksecurity_YYYYMMDD_HHMMSS.log`

## 开发相关

### Q: 如何添加新的机器学习算法？

修改 `networksecurity/components/model_trainer.py` 文件，在 `models` 字典中添加新算法。

### Q: 如何修改模型参数？

编辑 `config/config.yaml` 文件。

### Q: 如何贡献代码？

参考 [贡献指南](../CONTRIBUTING.md)。

## 错误处理

### Q: 出现 "ModuleNotFoundError" 错误？

确保已激活虚拟环境并安装了所有依赖：

```bash
source venv/bin/activate
pip install -r requirements.txt
```

### Q: 训练时出现 "FileNotFoundError"？

确认 `data/phisingData.csv` 文件存在。

### Q: 预测时返回500错误？

确保已经训练过模型，`models/` 目录下存在模型文件。
